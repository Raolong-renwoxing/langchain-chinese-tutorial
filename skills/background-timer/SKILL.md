# Background Timer Skill

演示如何在技能中后台运行脚本。接收倒计时秒数，在后台等待指定时间后，在工作目录创建一个包含随机字符的文本文件。

## 核心功能

1. 接收用户输入的秒数参数
2. 在后台启动倒计时脚本
3. 脚本执行完毕后，在工作目录创建随机字符文件
4. 通知用户任务完成

## 使用方式

当用户说类似以下内容时触发：
- "倒计时 50 秒"
- "后台运行 30 秒后创建文件"
- "等 60 秒然后生成随机字符串文件"
- "定时任务：120 秒后创建文件"

## 参数解析

从用户输入中提取秒数：
- "50 秒" → 秒数 = 50
- "1 分钟" → 秒数 = 60
- "倒计时30" → 秒数 = 30

如果用户未指定秒数，默认使用 10 秒。

## 执行流程

### 步骤 1: 解析参数

从用户输入中提取倒计时秒数。

### 步骤 2: 后台启动脚本

使用 RunCommand 工具在后台运行 Python 脚本：
先获取当前工作目录，然后执行命令：
```bash
python "<workspace>/background-timer/scripts/timer.py" <秒数>
```

**重要**：
1. 必须使用完整的绝对路径，不能使用相对路径
2. 必须使用 `blocking: false` 将任务放到后台运行，使其不阻塞当前会话

### 步骤 3: 通知用户

立即告知用户后台任务已启动，包括：
- 倒计时秒数
- 预计完成时间
- 将在哪个目录创建文件

### 步骤 4: 等待并确认

脚本运行期间不要主动查询状态。等待脚本执行完成后，确认文件已创建。

## 脚本说明

`scripts/timer.py` 是核心脚本：

- 接收 1 个参数：秒数
- 等待指定时间
- 生成随机字符（长度 32-64）
- 写入文件 `random_text_<timestamp>.txt` 到工作目录
- 输出完成信息

## 示例对话

**用户**: 倒计时 50 秒

**助手**: 收到！我将在后台启动一个 50 秒的倒计时任务。

⏱️ **后台任务已启动**
- 倒计时：50 秒
- 预计完成时间：约 50 秒后
- 输出文件：工作目录下的 `random_text_<时间戳>.txt`

（50 秒后）

✅ 任务完成！已创建文件 `random_text_20260319_143052.txt`，包含随机生成的字符串。

## 技术要点

1. **后台执行**：使用 `blocking: false` 将任务放到后台运行
2. **非阻塞**：后台任务不应阻塞 skill 的响应
3. **参数传递**：通过命令行参数传递秒数
4. **文件输出**：脚本自行决定输出位置（工作目录）
